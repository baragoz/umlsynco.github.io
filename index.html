Diagram.io is online UML editor for software architecure lovers !

Diagram.io is online UML editor that is oriented to Github. You could easily create a UML diagram in any modern Browser and commmit it to your repositoy on the GitHub. For the local clone of your Github repository you could use Eclipse plug-in which implementes WebSync HTTP API.

Project values:

  • Minimize the creation time for a UML diagram
  • Integrate diagrams into markdown documentation
  • Simplify sharing diagrams and documentation with collegues
  • Snippets - the way to share knowledge

Additional project values:

  • Framework for GitHub applications: It is a common task for all GitHub application developers to create a framework for the repository navigation, edit, commit etc. In scope of this project we have developed a framework for GitHub applications which could simplify an integration of any content editor:
    • Multiple files commit to GitHub;
    • API for a custom editor/viewer creation for the specific content type;
    • API to create a custom converter from one format to another which is acceptable by existing viewer; For example: UML OWL/RDF -> UML XMI;
    User could find and register any custom viewer (which is located on GitHub) and associate the content type with it.
  • HTTP API for the offline clone of repository: It is essential to do not create a PC application for the same web editor. Therefore we have developed a draft version of websync API which is similart to Eclipse DStore API and provides an access to the project tree, file-content and source code indexer API. Of course the base API should implement an autherntication procedure, file-tree navigation and file content request.
  • Integration of content into markdown documentation or another content viewer/editor: It is a common case when user needs to integrate some diagram into markdown document(at least for UML); In scope of this project we are thinking about the common way to provide the reference on content:
    • By full path
    • By relative path
    • By SHA sum
  • Widgets: It is often required to create an embedded viewer widget for the content integration on external site like www.stackoverflow.com; It could be code snippet, or UML diagram, or markdown documentation; So, one of the value of this project is to provide an easy way to create a viewer widgets for the user-specific content;
